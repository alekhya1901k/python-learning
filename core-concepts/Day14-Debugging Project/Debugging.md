Debugging means finding and fixing mistakes in code.

This project teaches 6 debugging techniques:

1. Describe the problem
2. Reproduce the bug
3. Play computer
4. Fix editor errors
5. Use print statements
6. Use a debugger

# Summary: Debugging Techniques

1. Always understand the problem before fixing code.

2. Reproduce the bug so you can test the fix.

3. Trace code line by line like the computer.

4. Check boundary values carefully.

5. Fix syntax and indentation errors first.

6. Use print() to inspect variable values.

7. Understand the difference between = and ==.

8. Use try/except for expected user input errors.

9. Use a debugger for deeper inspection.

10. Most beginner bugs come from ranges, indexes, indentation, and wrong operators.


# Best Beginner Debugging Flow

                Start
                ↓
                Read the problem
                ↓
                Run the code
                ↓
                Observe the error or wrong output
                ↓
                Describe what should happen
                ↓
                Check variables using print/debugger
                ↓
                Find the exact wrong line
                ↓
                Fix the code
                ↓
                Run again and test
