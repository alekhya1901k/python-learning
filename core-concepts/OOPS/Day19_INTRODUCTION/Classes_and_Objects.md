# 2. Classes and Objects
    
## Quick Summary Notes:
    1. A class is a blueprint for creating objects.
    2. An object is an instance of a class.
    3. Classes describe what objects have and do.
    4. Attributes represent object data.
    5. Methods represent object behavior.
    6. One class can create many objects.
    7. Class names usually use PascalCase.
    8. Object names usually use snake_case.
    9. Objects help model real-world entities.
    10. Classes improve code reusability.

# Class vs Object

## Class: Blueprint of a House

## Object: Actual House Built Using Blueprint

Example:

class Waiter:
    pass

henry = Waiter()
betty = Waiter()

Explanation
Waiter -> Class
Henry  -> Object
Betty  -> Object

Both objects are created from the same blueprint.
